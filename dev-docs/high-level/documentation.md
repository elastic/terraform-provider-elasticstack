# Documentation

Provider docs are generated; keep them in sync with code changes.

## Where docs come from

- Templates: `templates/`
- Examples: `examples/`
- Generated output: `docs/`

Canonical contributor guidance: “Updating Documentation” in [`contributing.md`](./contributing.md#updating-documentation).

## Terraform version used for docs generation

`make docs-generate` reads the Terraform CLI version pinned in the repository root `.terraform-version` file and passes that version to `tfplugindocs`.

This keeps docs generation deterministic across contributor machines and CI. Docs generation does not rely on whichever Terraform version happens to be installed locally.

The pinned version is maintained in `.terraform-version`. Renovate is expected to keep that file up to date over time.

## Regenerating docs

- **Standard provider docs:** `make docs-generate` (uses `TF_ELASTICSTACK_INCLUDE_EXPERIMENTAL=false`).
- **Full docs tree including committed experimental exceptions:** `make docs-generate-provider` — runs, in order:
  1. `make docs-generate`
  2. `make docs-generate-fleet-managed-integration`
- **Code + all docs:** `make gen` (runs `docs-generate-provider`, then `go generate ./...`).

`make lint`, `make check-docs`, and CI’s `gen` / `check-docs` steps use `docs-generate-provider` so the committed `docs/resources/fleet_managed_integration.md` is never left stale after default generation alone.

Note: `make lint` also runs `docs-generate-provider`.

If you change provider schemas, docs templates, or examples, regenerate with `make docs-generate-provider` (or `make gen`) and commit the resulting `docs/` updates.

### Fleet managed integration (experimental provider registration)

`elasticstack_fleet_managed_integration` is registered only when `TF_ELASTICSTACK_INCLUDE_EXPERIMENTAL=true`. Default `make docs-generate` does not include that resource in the provider schema, so its reference page is **not** produced by the standard pass alone.

The committed page is generated by the scoped target `make docs-generate-fleet-managed-integration`, which:

- Sets `TF_ELASTICSTACK_INCLUDE_EXPERIMENTAL=true` for that invocation only (does not publish other experimental entities to `docs/`).
- Uses the dedicated template under `fleet-managed-integration-docs/resources/fleet_managed_integration.md.tmpl` (outside `templates/` so the default pass does not try to render it).

Practitioners must set `TF_ELASTICSTACK_INCLUDE_EXPERIMENTAL=true` for Terraform plan/apply against that resource; the generated doc states this explicitly.
