## Why

Running multiple parallel AI agents requires each worktree to have an isolated Elastic Stack (Elasticsearch, Kibana, Fleet) — but the repo currently uses hardcoded container names and ports, meaning any two worktrees starting `docker compose up` would conflict. Worktrunk is already installed and the repo is already structured as a bare repo with worktrees; formalising the configuration removes manual setup friction and makes isolated parallel development reliable.

## What Changes

- Add `.env.template` (committed): static docker/test configuration without container names or ports
- Add `.config/wt.toml` (committed): worktrunk project hooks — `pre-start`, `post-start`, `pre-commit`, `pre-remove`
- Remove all `container_name:` directives from `docker-compose.yml` so Docker Compose namespaces containers and volumes by project (worktree directory name) automatically
- Remove container name variables from `.env` / `.env.template`
- Update `Makefile`: add `ELASTICSEARCH_PORT ?= 9200` and `KIBANA_PORT ?= 5601` defaults; replace hardcoded `localhost:9200` / `localhost:5601` literals in affected targets with `$(ELASTICSEARCH_PORT)` / `$(KIBANA_PORT)`
- Document user config in `dev-docs` (worktree path template, shell integration)

## Capabilities

### New Capabilities

- `worktrunk-project-config`: Worktrunk project configuration (`.config/wt.toml`) and supporting files (`.env.template`) enabling isolated per-worktree Elastic Stack environments and automated worktree lifecycle hooks.

### Modified Capabilities

*(none — no existing specs cover docker-compose or Makefile port configuration)*

## Impact

- `docker-compose.yml` — container name directives removed
- `.env` — gitignored; now generated by `post-start` hook rather than manually maintained
- `.env.template` — new committed file replacing the manually-kept `.env`
- `.config/wt.toml` — new committed file
- `Makefile` — port variable defaults added; ~6 hardcoded localhost targets updated
- Developer workflow: new worktrees get a generated `.env` with unique ports automatically; `make setup` runs on creation; `make check-lint` runs before every commit; docker stack tears down when a worktree is removed
